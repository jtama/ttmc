À quoi sert la balise <div> en HTML ?

Elle sert à structurer le contenu en blocs.

Quelle est la différence entre les balises <span> et <div> ?

<div> est un élément bloc (il prend toute la largeur).
<span> est un élément inline (il s’intègre dans une ligne de texte).

À quoi sert l’attribut srcset sur une balise <img> ?

Il permet de fournir plusieurs versions d’une image (différentes résolutions), et le navigateur choisit la plus adaptée (responsive images).

Qu’est-ce que le shadow DOM (lié à HTML) ?

Le Shadow DOM est une fonctionnalité du Web Components API qui permet de créer un DOM encapsulé (ou isolé) à l’intérieur d’un élément HTML personnalisé.

Que permet la balise <template> en HTML5 ?

Elle définit du contenu HTML qui n’est pas affiché au chargement, mais peut être cloné via JavaScript.

Quelle est la différence entre les attributs aria-* et les balises HTML sémantiques pour l’accessibilité ?

Les balises sémantiques (comme <header>, <nav>) donnent une structure native accessible. Les attributs aria-** complètent ou corrigent quand la sémantique native ne suffit pas. Ensemble, ils améliorent l’accessibilité (lecteurs d’écran, navigation clavier).

Quelle est la différence entre <iframe> et <object> ?

<iframe> intègre un document HTML externe. <object> est plus générique (peut embarquer HTML, images, PDFs, etc.) et fait partie des balises multimédia.

Pourquoi l’attribut rel="noopener noreferrer" est-il recommandé sur les liens externes avec target="_blank" ?

Pour éviter les failles de sécurité (window.opener exploitable).

Que permet de faire la balise <picture> en HTML5 et quelle est sa différence avec srcset ?

<picture> permet de définir plusieurs sources (<source>) avec conditions (media queries, formats comme WebP). Plus puissant que srcset, il permet de changer d’image selon contexte et pas seulement résolution.

Comment le navigateur construit-il l’accessibility tree à partir du DOM et du CSSOM ?

Cet arbre n’est pas identique au DOM visuel mais reflète la structure logique et accessible.

  1. Il part du DOM.
  2. Applique les styles sémantiques et ARIA
  3. Ignore les éléments non pertinents (display: none, aria-hidden="true").
  4. Génère un arbre séparé pour les technologies d’assistance (lecteurs d’écran).
